PRACTICAL ONE MATLAB Simulink

Aim

To successfully load, configure, and evaluate an external Fuzzy Inference System (.fis) block within a MATLAB Simulink simulation loop architecture.

Objective
  • Design and export a valid Fuzzy Inference System definition matrix directly into the MATLAB global workspace memory footprint.
  • Incorporate an operational Fuzzy Logic Controller block into an existing Simulink signal-routing canvas.
  • Link the controller block dynamically to the workspace parameters to drive downstream signal outputs safely based on incoming input variations.
Methodology

The workspace environment was initialized by invoking the fuzzy design application panel via the command line fuzzy. Inputs, membership shapes, and conditional rule arrays were configured before exporting the structure directly to the workspace under a dedicated variable name. Simulink was then opened by running simulink. A structural logic model was created using a Step Input block, a Scope visualization layout, and the native Fuzzy Logic Controller block dragged from the Fuzzy Logic Toolbox library. Double-clicking the controller block opened its parameter configurations, where the exact global variable string matching the exported workspace FIS structure was specified.

Observation

Upon initializing execution cycles, the Simulink simulation engine fetched properties from the designated workspace variable without encountering dimensional structural errors. The scope accurately plotted smooth, nonlinear corrective control steps instead of basic step responses.

Conclusion

Importing `.fis` structures into Simulink streamlines the transition from descriptive linguistic rule arrays to dynamic, closed-loop time simulations, simplifying verification processes for intricate engineering problems.

PRACTICAL TWO FIS Comparison

Aim

To evaluate and compare the deployment workflows of constructing a Fuzzy Inference System via the interactive MATLAB Graphical User Interface vs. hardcoded programmatic scripts (.m file).

Objective
  • Build a fully functional multi-input Mamdani Fuzzy System using interactive GUI Toolbox panels.
  • Replicate the exact variable spaces, linguistic conditions, and rules programmatically via foundational MATLAB terminal command structures.
  • Generate and compare 3D output surfaces from both implementations to confirm algebraic execution parity.
Methodology

The visual framework was created via the GUI toolboards, specifying boundaries, setting up triangular membership groupings, and linking inputs through the built-in rule builder matrix. For the code-driven setup, an explicit script execution flow was written within a native .m file using structured matrix functions: mamfis() to create the system container, addInput() to define tracking channels, and addRule() to populate the logic map. The computed output response spaces were then visualized side-by-side using the gensurf() plotting command.

Observation

The GUI layout allowed for rapid adjustments to coordinate intersections, whereas the script-based approach excelled at batch variations and rapid parameter changes. Both methods generated identical mathematical response spaces when evaluating identical rule sets.

Conclusion

The GUI Toolbox serves as an excellent prototyping environment for initial design layouts, while script-driven generation is much better suited for high-throughput optimizations and embedding logic within algorithmic loops.

PRACTICAL THREE MATLAB Matrix Operations

Aim

To execute matrix calculations inside the MATLAB Command Line environment, emphasizing the functional differences between linear algebraic operations and element-wise array math.

Objective
  • Instantiate multi-row reference array classes $A$ and $B$ with explicit tracking values.
  • Compute the matrix addition ($A + B$) to verify coordinate-wise cell evaluations.
  • Execute element-by-element array multiplication ($A .* B$) to evaluate performance against standard cross-product rules.
Methodology

The designated tracking dimensions were typed directly into the MATLAB terminal window buffer line queue:

A = [1 2 3; 4 5 6; 7 8 9];
B = [4 5 6; 7 8 9; 10 11 12];
result_addition = A + B;
result_element_multiplication = A .* B;
Observation

The matrix addition operation calculated basic cell-to-cell values directly across matching indexes. For multiplication operations, omitting the dot modifier (A * B) triggered standard algebraic row-by-column dot products, whereas appending the dot prefix operator (A .* B) scaled individual elements independently at matching coordinates.

Conclusion

Understanding the distinction between standard matrix algebra and element-wise dot-notated modifiers is crucial for ensuring accuracy when mapping multidimensional transformations or tensor processing routines.

PRACTICAL FOUR Python Linear Regression

Aim

To design, optimize, fit, and validate a supervised Linear Regression Machine Learning model using Python's scikit-learn library tools.

Objective
  • Generate structured independent dataset arrays ($X$) and map corresponding continuous dependent values ($y$).
  • Train the predictive model using the LinearRegression fitting framework to locate the optimal global trend line.
  • Extract performance metrics, including the Coefficient of Determination ($R^2$) and Mean Squared Error (MSE), to evaluate trend accuracy.
Methodology

A Python execution environment was set up by importing numpy, matplotlib.pyplot, and LinearRegression from sklearn.linear_model. The sample dataset array vectors were defined, and the structural features were reshaped into a multi-row dimensional layout. The system model parameters were optimized by running the .fit(X, y) learning routine. Predictions were generated across the target range, and a visual chart was rendered using the plt.scatter() and plt.plot() functions to display coordinate metrics.

import numpy as np
import matplotlib.pyplot as plt
from sklearn.linear_model import LinearRegression

X = np.array([1, 2, 3, 4, 5]).reshape(-1, 1)
y = np.array([2, 3.9, 6.1, 8.0, 10.2])

reg_model = LinearRegression().fit(X, y)
y_pred = reg_model.predict(X)
Observation

The training routine converged quickly, establishing an optimal slope and intercept that minimized residual variance. The evaluation metrics confirmed strong predictive alignment, showing a high $R^2$ score close to 1.0 and a minimal residual Mean Squared Error.

Conclusion

Supervised linear regression models provide a highly reliable framework for predictive analytics when working with highly correlated continuous variables.
